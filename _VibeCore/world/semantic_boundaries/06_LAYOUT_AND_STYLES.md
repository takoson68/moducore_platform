# Layout & Styles — World Semantic Addendum

本文件補完「佈局（Layout）與樣式（Styles）」在**世界層（World Layer）**中的語意定位。  
本文件**不描述設計實作、不定義技術細節、不推導樣式策略**，僅界定世界語意中各層級所承擔的責任與邊界。

---

## 1. 佈局層的世界語意角色

Layout 是世界內容的**呈現框架（Presentation Frame）**。

在世界語意中，Layout 的責任僅包含：

- 提供一致且穩定的視覺容器
- 承載世界已成立的內容與狀態

並受以下語意限制：

- ❌ Layout 不承擔世界裁決
- ❌ Layout 不負責權限判斷
- ❌ Layout 不管理能力或狀態
- ❌ Layout 不產生或修正世界語意

Layout **只能反映世界狀態**，而不能塑造世界狀態。

---

## 2. 樣式分層的語意邊界

樣式分層存在的語意目的在於：

- 維持世界的一致性（Consistency）
- 保留世界結構的可替換性（Replaceability）

而**非**追求單一模組的視覺效果或工程便利。

在世界語意中，樣式層級必須保持以下邊界清晰：

- **全域樣式**：定義世界層級的共同視覺語彙
- **專案樣式**：承載 Project 特定但已被世界承認的差異
- **模組樣式**：僅影響模組自身的呈現

明確禁止：

- ❌ 模組以樣式行為影響世界裁決
- ❌ 模組以樣式改變其他模組行為
- ❌ 利用樣式層級製造跨模組副作用

樣式是**呈現手段**，不是**語意工具**。

---

## 3. 可替換性與世界一致性

佈局與樣式必須具備可替換性，但此可替換性：

- 必須服從世界既有規則
- 不得破壞世界結構的一致性
- 不得造成語意上的混淆或漂移

在語意上明確禁止：

- 任何樣式策略造成跨 Project 的語意混用
- 以樣式選項隱性改變世界行為或邊界

世界結構的穩定性，高於任何視覺自由度。
