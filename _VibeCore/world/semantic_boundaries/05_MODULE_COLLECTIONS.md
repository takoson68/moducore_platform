# Module Collections — World Semantic Addendum

本文件補完「模組集合（Module Collections）」在**世界層（World Layer）**中的語意定位。  
本文件**不定義工程集合方式、不描述載入策略、不推導實作手段**，僅界定世界語意中「模組集合」所承擔的責任與邊界。

---

## 1. 模組集合的世界語意

模組集合是一種**世界層的宣告結果（Declarative Outcome）**，  
表示世界針對某一 Project 所**正式承認的可用模組範圍**。

在世界語意中適用以下原則：

- 模組集合**不是**模組的實體清單
- 模組集合**不是**工程掃描或載入結果
- 世界只承認**被合法宣告的集合**
- ❌ 世界不推論、補齊或假設未宣告模組的存在

模組是否可用，來自**集合裁決**，而非檔案結構或程式存在。

---

## 2. 模組集合與裁決權的分離

模組集合的成立，必須由**世界裁決**與 **PlatformConfig** 共同限定。

在語意邊界上：

- 世界裁決定義「允許的集合範圍」
- PlatformConfig 提供「可被選取的集合配置」
- 工程層僅負責**承接並落實已成立的集合**

明確禁止以下行為：

- ❌ 工程層擅自擴張模組集合
- ❌ 以實作便利產生隱性或未宣告集合
- ❌ 透過條件判斷、動態匯入暗中改變集合語意

集合是**被裁決的結果**，不是工程自由度。

---

## 3. 集合的可替換性與隔離性

模組集合在世界語意中必須具備：

- **可替換性（Replaceability）**  
  集合可以被整體替換，而非逐模組漂移變動
- **隔離性（Isolation）**  
  集合僅對其所屬 Project 世界成立

為維持世界單一性與可預期性，適用以下語意限制：

- 集合的變動不得破壞既有世界語意
- 集合的切換不得造成世界狀態混疊
- ❌ 任何跨 Project 的模組集合共享，在語意上均屬非法

世界只存在**一個有效集合上下文**。
