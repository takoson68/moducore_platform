# World Model Extension Index

本目錄為世界主鏈（00–08）之延伸敘事層，
用以承載世界中既有概念的「可結構化描述」。

本目錄不構成新的世界層，
亦不具備獨立的世界定義權限，
其語意地位從屬於世界主鏈。

---

## Purpose of This Layer

model 層的存在目的為：

- 將世界主鏈中已被承認存在的概念
  轉化為可被選擇、判斷與組裝的結構描述
- 提供世界在進入生成、組裝或判定前
  所需的最低語意完整度
- 作為世界理解與實際配置之間的中介層

model 層不回答「世界是什麼」，
只回答「世界如何承認一個存在被宣告」。

---

## Scope and Boundary

本目錄文件僅限於以下行為：

- 補充世界主鏈中既有概念的結構輪廓
- 描述其被世界承認時所需的最低條件
- 定義世界在組裝與判斷時的結構邊界

本目錄文件**明確不負責**：

- 新增或修改世界規則
- 定義任何流程、時序或生命週期
- 描述執行行為或實作細節
- 取代或重寫世界主鏈中的語意判斷

若某內容改變了對世界的理解，
則該內容不得存在於本目錄。

---

## Reading Rules

請依下列原則決定是否進入本目錄：

- 若你正在理解「世界的構成與假設」  
  → 請閱讀 `ai/world` 目錄中的主鏈文件（00–08）

- 若你需要判斷：
  - 一個 Project 是否被合法宣告
  - 一個 Module 是否構成世界可承認的單位
  - 不同存在的組合是否可能成立  
  → 再進入本目錄閱讀對應文件

本目錄不具備固定閱讀順序，
其使用時機由任務需求決定。

---

## Model Chain Overview

目前 model 層文件形成一條線性敘事鏈，
其順序具有語意意義：

1. `01_PROJECT_CONFIG.md`  
   - 定義 Project 作為可生成存在的最低宣告條件

2. `02_MODULE_DEFINITION.md`  
   - 定義世界承認之 Module 存在語意

3. `03_PLATFORM_CONFIG.md`  
   - 定義多 Project 所共享的世界上下文宣告

4. `04_ASSEMBLY_CONSTRAINTS.md`  
   - 定義世界明確拒絕的結構性組合

5. `05_CONSISTENCY_RULES.md`  
   - 定義文件之間出現語意衝突時的裁決依據

後續文件若加入，
必須能明確說明其在此敘事鏈中的位置與責任。

---

## Extension Discipline

任何新增至本目錄的文件，
必須同時符合以下條件：

- 所描述內容已在世界主鏈中被承認存在
- 該內容不會改變世界對該概念的理解
- 若移除此文件，世界仍可被理解，但難以被使用
- 該文件之語意責任可被清楚界定於 model 層

若不符合上述條件，
該內容不得納入 model 層。

---

## Final Note

本目錄的存在目的，
不是讓世界更大，
而是讓世界 **更可被正確使用**。

當世界理解與實際配置之間出現落差時，
本目錄提供補足；
但當補足導致新的理解歧義時，
世界主鏈永遠具有最終解釋權。
