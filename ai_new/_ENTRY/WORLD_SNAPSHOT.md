# 世界縮影（World Snapshot）

這個世界本質上是一個以「世界法則」為核心的專案生成平台，而不是單一程式庫或一般專案。它不是為了完成某一個功能，而是為了在同一套可裁決、可維持一致性的世界規則下，反覆生成不同的 Project。與一般專案追求單一交付不同，這個世界追求的是可被重複成立、可被替換、可被驗證的結構一致性。它存在的理由是為了讓能力可模組化、可插拔、可替換，而不因一次工程決策而破壞整體世界的可預期性。「不改世界生成專案」不是技巧，而是世界存在方式的核心，因為只有在不改變世界的前提下生成不同 Project，世界才保有穩定性與可裁決性，這也是世界能夠被長期運作與理解的根基。

因此，世界必須具備單一且封閉的裁決來源，否則世界將無法保有一致的邊界與語意。這個裁決權唯一來自 ai_new/world，世界主鏈文件構成唯一權威，且閱讀順序由 INDEX.md 指定，確保理解與裁決不被片段化或被工程便利改寫。API World 與 Module World 只是延伸層，僅承載語意描述，不具裁決權，不能取代或改寫世界核心。當裁決權不再單一或不再封閉，世界就會失去作為「可被治理的世界」的本質，而退化為任意拼裝的工程集合。

工程在這個世界中被允許存在，是因為世界需要可被落地的工作方式與實作行為，但工程只是一種在世界邊界內工作的角色，而不是裁決者。工程的價值在於把世界規則變成可被執行的實作，維持結構一致性與可追溯性，讓世界能在不同 Project 中被有效重現與驗證。工程不得成為裁決者，因為裁決屬於世界核心，工程若自行裁決，將使世界規則變成可被隨意改寫的工具，破壞世界的可預期性與一致性。工程的定位是服從世界、維護世界，而不是替代世界。

因此，限制、紀律與回退行為不是為了限制行動，而是為了保護世界的可裁決性與可理解性。任何未被世界明確允許的行為，都必須被視為不合法並立即停止，因為一旦工程凌駕世界，世界就不再穩定。遇到不確定、模糊或歧義時，唯一正確的行為是停止工程並回報世界層問題，不能自行補完或推論。問題要修正或記錄於 feedback，工程行為必須可追溯，未被記錄視為不存在，這些約束確保世界能被長期維持，而不是被短期便利侵蝕。
